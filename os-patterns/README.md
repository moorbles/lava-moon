# OS Patterns

Cross-brand learnings distilled from the insight corpus. This folder is the asset the project exists to build. Brand 1 is the test vehicle. What proves true here, and later across Brand 2, graduates into a reusable pattern for launching any DTC brand.

## What belongs here
Not brand-specific insights. Those live in `brands/<brand>/`. This folder holds the generalised version: the pattern that held regardless of the specific product.

Graduation example:
- `brands/noomanara/crm/...-timeline-education-sequence.md` is Noomanara-specific (days 3/14/30/60 for a slow-onset sleep stack).
- The OS pattern is the generalisation: any DTC product with delayed felt effect needs a sequenced education flow that reframes the silent period, mapped to the product's mechanism timeline. It graduates here once it holds on a second brand.

## Candidates already visible from Brand 1
- Supplier qualifying template (SUP-004): a brand-agnostic manufacturer-qualifying checklist.
- Machine-vs-product gate (DEC-004): a transferable pass/fail scaffold for any single-brand validation.
- Fix first-order with AOV not retention (FIN-001): a DTC unit-economics principle.

Do not pre-write these from one brand's data. A pattern earns its place by holding more than once.

## Format
Same schema as `templates/insight.md`, with `type: os-pattern` and `brand: all`.
